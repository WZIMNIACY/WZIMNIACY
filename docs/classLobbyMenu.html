Detailed Description

Główny ekran lobby odpowiedzialny za zarządzanie listą graczy, drużynami, trybem gry, typem AI, kluczem API oraz rozpoczęciem sesji gry. Integruje sygnały i wywołania z EOSManager, korzysta z detekcji sprzętu (Diagnostics.HardwareResources) oraz pokazuje podpowiedzi przez CustomTooltip.

Zakłada obecność autoloadu EOSManager oraz zainicjalizowanych węzłów sceny (przypięte exporty). Operacje powinny być wywoływane w wątku głównym Godota; nie jest thread-safe.

◆ _Ready()

override void LobbyMenu._Ready ( )
inline

Inicjalizuje referencje, podpina sygnały UI oraz EOSManager, wczytuje tooltips i wykonuje początkowe odświeżenie stanu lobby (drużyny, tryb gry, AI, klucz API).

Ustawia początkowe połączenia sygnałów tylko dla dostępnych węzłów oraz inicjuje detekcję VRAM w tle.

Loguje błąd, gdy autoload EOSManager jest niedostępny lub scena nie ma aktywnego lobby.
